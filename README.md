## Overview

VillageVerse is a full-stack cozy island-life simulation. Players browse and buy items in a themed marketplace, gather resources at a pirate camp to upgrade it over time, and build a login streak for daily coin rewards вҖ” all backed by a real, persistent FastAPI + SQLAlchemy backend rather than mock data.

---

## Features

| Feature | Description |
|---|---|
| рҹӣҚпёҸ **Live Shop** | 33-item catalog across 13 categories, real coin-based buying, item preview, theme filtering |
| рҹҸ•пёҸ **Pirate Camp** | Gather wood/stone/cloth from resource nodes on cooldowns, spend them to upgrade camp level |
| рҹҺҒ **Daily Rewards** | Streak-based login rewards that grow the longer you keep your streak alive |
| рҹҺЁ **3 Themes** | Cozy Day, Starlight Night, Pirate Camp вҖ” switchable from a header menu, persisted per-browser |
| рҹ“ұ **Swipe Navigation** | Mobile swipe-to-navigate between pages |
| вҡЎ **Async Backend** | FastAPI + SQLAlchemy async engine, rate-limited via SlowAPI |

## Quick Start

```bash
# 1. Clone the repository
git clone https://github.com/Kaweri05/VILLAGEVERSE-cozy-island.git
cd VILLAGEVERSE-cozy-island

# 2. Backend setup
cd backend/villageverse-backend
pip install -r requirements.txt
uvicorn app.main:app --reload
# вҶ’ runs at http://127.0.0.1:8000
# вҶ’ interactive docs at http://127.0.0.1:8000/docs

# 3. Frontend setup (in a second terminal)
cd frontend
npm install
npm run dev
# вҶ’ runs at http://localhost:3000
```

Keep both terminals running at the same time вҖ” the frontend calls the backend directly over HTTP.

## Environment Variables

Backend `.env` (in `backend/villageverse-backend/`):

| Variable | Required | Description |
|---|---|---|
| `DATABASE_URL` | Yes | `sqlite+aiosqlite:///./villageverse.db` locally, or a Postgres connection string in production |
| `DATABASE_ECHO` | No | `true` to log all SQL statements |
| `JWT_SECRET` | Yes | Secret key for future auth work |
| `JWT_ALGORITHM` | No | Defaults to `HS256` |
| `JWT_EXPIRE_MINUTES` | No | Defaults to `60` |

In Vercel, set `DATABASE_URL` under **Project Settings вҶ’ Environment Variables** to your hosted Postgres (Neon/Supabase) connection string, using the `postgresql+asyncpg://` prefix.
